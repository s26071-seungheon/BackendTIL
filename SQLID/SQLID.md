# SQLID
 - 데이터베이스에 SQL문을 쓰면 DB는 긴 문장을 해석하기 힘들어 Hash함수 이용해 고유한 문자열(식별자)를 만들어내는 것
   - Hash함수 : 복잡한 데이터를 입력받더라도 일정한 길이의 짧은 코드로 요약해주는 함수
 - 쿼리 내용이 한 글자(공백포함),대소문자, 다르면 SQLID가 바뀜
 -  ex) aaa, a aa, Aaa
 - 문자열 형태는 알파벳과 숫자가 섞인 13자리 형태
   - 8fjk2s91asdf 이런식
 - 바코드, 디지털 지문, 죄수번호 같은 것이라 생각

 ## SQLID 핵심 역할
 - 라이브러리 캐시 최적화
   - DBMS는 SQL을 실행하기 전 어떻게 실행할 지 계산하는 비용이 큼
   - SQLID를 통해 이미 실행된 적이 있는 쿼리인지 빨리 판단하여 기존에 계산해 둔 실행 계획을 재사용하여 DB부하 줄임
 
 - 커서 공유 및 매핑
   - 라이브러리 캐시 내에서 각 SQL 문장의 주소록 역할
   - 특정 SQL의 통계 정보, 실행 횟수, 사용된 리소스 정보 등을 이 SQLID에 매핑하여 관리
   - SQLID(Key)를 검색하면, 해당 쿼리의 실행 계획이 담긴 커서(Value)의 위치를 바로 알 수 있고, 덕분에 똑같은 쿼리를 실행하려는 다른 사람들과 그 정보를 공유함

 - 성능 모니터링 및 추적
   - 수만 개의 쿼리중 시스템에서 느린 쿼리를 특정할 때 SQLID를 사용
   - ASH로 SQLID를 1초마다 찍어 기록
     - 예를 들어 1시 10분에 느렸다면 1시 10분에 찍힌 SQLID바로 찾음
   - AWR로 하루동안 가장 느렸던 SQLID 순위 매겨 보고서 형식으로 뽑아줌
     - ex) 1. SQLID: 8fjk2s, 2. SQLID: 9abc3z
