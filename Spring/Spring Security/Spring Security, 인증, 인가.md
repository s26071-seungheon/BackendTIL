# Spring Security
 - 우리 서비스에 누가 들어올 수 있고, 들어와서 무엇을 할 수 있는지를 관리하는 보안 프레임워크

# 핵심 개념
 - 인증 (Authentication)
   - 로그인 절차가 이에 해당 사용자가 누구인지 확인하는 과정(아이디/비밀번호)
 - 인가 (Authorization)
   - 들어가서 뭘 할 수 있는 지 권한을 제한 하는 것
 - 인증과 인과의 차이점
   - 인증은 자기가 누구인지 확인하는 것과 같고 인가는 나의 권한이 어디까지 있는 지 확인 하는 것
 
# Spring Security의 동작원리
   - 필터들의 집합으로 동작
     - ex) 검문소1, 검문소2, 검문소3 이렇게 검문소같이 여러겹으로 나눠져 최종통과를 해야 컨트롤러 로직이 실행

# 쓰는 이유
   - 표준화된 보안
     - 내가 만든 것보다 나보다 잘하는 사람이 만든 게 더 안뚫리니까
   - 기본 적인 해킹 공격 알아서 막아줌
