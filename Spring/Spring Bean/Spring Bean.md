# Spring Bean
 - 스프링이 컨테이너에 담아두고 직접 관리하는 객체
 - 쉽게 이해하면 new 키워드 대신 사용하는 것

## 일반객체와 Spring Bean의 차이
 - 가장 큰 차이는 누가 관리하는가임
 - 일반객체는 자신이 필요할 때 내가 new 키워드로 직접 만듦
 - Spring Bean은 스프링이 스스로 객체 생성, 필요한 곳에 넣고, 끝까지 관리
   - 개발자가 Spring한테 이 객체 관리해줘 라고 말만 해주면 관리해줌

## 굳이 Spring Bean을 쓰는 이유
 - 의존성 주입(DI)
   - 어떤 부품이 필요한지 내가 말만 하면 Spring이 조립을 대신 해줌
 - 제어의 역전(CI)
   - 객체를 생성부터 끝까지 스프링이 관리하므로 개발자는 다른 것에 더 신경 쓸 수 있음
 - 싱글톤(Singleton) 관리
   - 스프링은 빈을 딱 하나만 만들고 여러곳에서 객체 하나를 돌려쓰기 때문에 메모리가 절약됨

## 일반객체를 쓰는 경우
- 데이터를 담는 객체 즉, 참조값이 달라야하는 객체는 일반 객체(new)를 씀

## 스프링 빈을 쓰는 경우
- 기능만을 수행하는 객체들 참조값이 같아도 되는 객체를 만들 때 쓴다.
- 아까 말한 싱글톤 관리때문에 데이터를 담는 객체에다 써버리면 데이터 다 섞여버린다.
