# 어노테이션
 - 자바 코드에 붙이는 '마크(표시)'이자, 컴퓨터(프레임워크)가 읽는 특별한 명령어
 - ex) 세탁기를 새로 샀는데 세탁기 버튼 옆에 포스트잇 붙여놈
   - 포스트잇 내용: 버튼을 3초동안 눌러야함
   - @(어노테이션)은 이 포스트잇을 읽고 내용안의 동작 방식으로 바꿔 버리는 것
 
# 어노테이션의 역할
 - 메타데이터의 제공
   - 코드에 대한 설명서 + 코드를 함께 전달
   - ex) 이 클래스는 컨트롤러라는 성질을 가졌어라고 정보 선언
 - 비침투적 설정
   - 소스 코드 본연의 로직을 건드리지 않고 외부에서 동작을 제어할 수 있게 만드는 역할
   - 로직에 집적 넣는 게 아니라 표식만 남기고 프레임워크에서 이를 읽고 처리
   
# 어노테이션의 장점
 - 코드의 가독성과 생산성이 향상됨
 - 비즈니스 로직과 설정의 분리
   - DB/연결/커밋/예외 처리 같은 코드 직접 짤 필요 없음 @기능 딸깍이면 됨
 - 컴파일 타임의 안전성 확보
   - @Override같이 오타 등으로 인한 재정의 실수를 컴파일 단계에서 즉시 잡아내어 프로그램의 안정성을 높여줌
