# JWT(JSON Web Token)
 - 서버가 부담 줄이기 위해 사용자에게 발급해주는 전자 신분증같은 느낌
   - 이럼 서버는 내가 준 신분증이 맞는지만 확인하면 됨

# JWT를 쓰는 이유
 - 세션방식은 내가 다 기억해야하는데 그럼 감당하기 힘듦
   - 세션 : 서버가 손님정보를 다 적어놓고 관리하는 방식
 - JWT방식은 서버는 자기가 준 신분증인지만 확인하면 됨

# JWT의 생김새
 - JWT는 "."을 기준으로 세 부분으로 나눔(aaa.bbb.ccc) 이런식
 - Header : 이건 JWT이고 암호화 알고리즘은 무엇을 썼어라고 알려줌
 - Payload : 진짜 데이터가 들어있음 사용자 아이디, 권한, 만료시간 등
 - Signature : Header,Payload를 합친다음 서버만 알고있는 비밀키를 더해 암호화시킨 결과물
    - Signature는 비밀키로 권한 같은 것을 고칠 수 없게 함

# JWT의 장점과 단점
 ## 장점
 - 확장성이 좋아 서버 여러대 여도 토큰만 검사하면 끝
 - HTTP 헤더에 담아서 보내기만 하면 됨
 ## 단점
 - 한 번 발금하면 취소 어려움
 - Payload는 암호화된 게 아니라 인코딩 되어있어 개방되어있음.
