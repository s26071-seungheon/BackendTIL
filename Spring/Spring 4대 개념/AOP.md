# AOP(Aspect Oriented Programming)
 - 관점 지향 프로그래밍
 - 공통기능을 모아 한 번에 관리

 ## AOP의 역할&장점
 - 관심사의 분리
   - 코드를 핵심관심사와 공통관심사로 나눔
     - 핵심 관심사 : ex) 입금, 출금, 송금같은 실제 로직
     - 공통 관심사 : ex) 로그 남기기, 보안체크같은 무조건 들어가는 부가 기능
 - 코드 중복 제거
   - ex) 여러개 메서드에 실행시간을 넣을 때 한 곳(Aspect)에만 측정 코드 작성하고 적용하라고 설정
 - 유연한 기능과 삽입 제거
   - ex) 로그가 필요하다 필요가 없어졌을 때 만들었던 한 개의 규칙에만 삭제가능

## AOP의 구성요소
 - Aspect : 공통기능 묶음
 - Advice : 실제 수행할 코드
 - Join point : 끼어들 수 있는 지점들
 - Pointcut : 적요할 대상을 지정
 - Weaving : 실제 결합하는 과정
