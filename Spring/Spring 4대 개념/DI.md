# DI(Dependency Injection)
 - IoC를 실제로 구현하는 방법
 - 객체를 스스로 생성하지 않고 외부에서 주입 받는 것

## DI의 주요 역할
 - 의존성 전달 : 필요한 객체를 외부에서 안으로 넣어줌
 - 연결 고리 해제: 객체 내부의 new 키워드를 제거하여 객체 간의 직접적인 연결을 끊는 역할
 - 제어권 위임: 객체 생성의 주도권을 개발자나 객체 자신으로부터 빼앗아 컨테이너로 넘기는 역할

## DI의 3대 요소
 - 사용자(Client) : 의존 객체를 사용하는 객체
 - 서비스(Service) : 사용자가 필요로 하는 의존 객체
 - 주입자(Injector) : 사용자의 생성자, 메서드 호출해 서비스를 주입 해주는 외부존재

## DI의 장점
  - 객체 간 결합도 낮춤
    - 객체 간 결합도 : 한 객체가 다른 객체에 의존하고 있는 정도
    - 의존도가 낮으면 코드 바꾸기가 쉬움
  - 객체의 수동적 관계 형성 (IoC 실현)
      - 객체는 자기가 어떤 클래스의 메서드가 실행될 지 모름
      - 객체는 자기가 사용 할 클래스의 메서드를 기다림
  - 테스트 용이성
    - 단위 테스트 할 때, 복잡한 객체 생성 X
    - 가짜 객체 주입하여 해당 클래스 로직만 검증 가능

## IoC와 DI의 차이
 - IoC가 생각이라면 DI가 그 생각을 실천하기 위한 행동
 - IoC는 @Component, @Service같은 어노테이션 사용해서 스프링에게 권한 넘긴 상태
 - DI는 권한을 받은 스프링이 @Autowired나 생성자를 통해 객체와 객체를 연결 해주는 것
