# IoC(Inversion of Control)
 - 개발자가 객체를 생성(new)하고 관리가 아니라 스프링에서 프레임 워크 또는 스프링 컨테이너가 생성, 관리
 ## IoC의 주요 역할
 - 객체 생성 : new 키워드 대신 컨테이너가 자바 객체 생성
 - 의존성 설정 : 객체끼리 서로 필요한 존재끼리 연결
 - 객체의 관리 : 객체 초기화부터 소멸까지 과정을 컨테이너가 담당

 ## IoC의 핵심 구성 요소 
 - IoC 컨테이너 : 객체 생성과 관리, 의존성(DI) 주입
   - BeanFactory : 스프링 설정 파일에 등록된 빈 생성 관리 컨테이너
   - ApplicationContext : BeanFactory의 기능 상속 + 부가기능
 
 - Bean : 스프링 컨테이너에 관리되는 자바 객체
   - new로 만들어진 객체가 아닌 것
   - @Component가 스프링에게 관리해야할 빈이라는 것을 알려줌
 ## IoC의 장점
 - 결합도 감소 : 인터페이스 기반 설계하면 구체적인 구현 클래스 변경해도 로직 변경 X
 - 단일 책임 원칙 준수 : 객체는 로직에만 집중, 객체 생성, 관리는 컨테이너가 담당
 - 코드 유지보수성 향상 : 시스템의 전체 구성 간단 변경
